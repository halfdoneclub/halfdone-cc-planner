# 에이전트 설계서 작성 가이드 (지식 파일)

> Tier 3에서 에이전트 설계서를 작성할 때 참조하는 가이드.
> 판단/코드 분리, 검증 패턴, 실패 처리, 에이전트 구조, 폴더 구조 등 설계 원칙을 포함.

---

## 에이전트 설계서 구조

```
# [작업명] 에이전트 설계서

## 1. 작업 컨텍스트
- 배경, 목적, 범위
- 입력 / 출력 정의
- 제약조건
- 용어 정의 (필요 시)
- 완료 조건: 전체 워크플로우가 "끝났다"고 판단하는 기준

## 2. 워크플로우 정의

### 단계별 흐름
| 단계 | 설명 | 담당 | 입력 | 출력 | 성공 기준 | 실패 시 |
|------|------|------|------|------|----------|--------|
| 1    |      | AI/코드 |  |  |  | 재시도/에스컬레이션/스킵 |

### 분기 조건
- 조건 A이면 → X단계로
- 조건 B이면 → Y단계로

### 단계 간 의존성
- N단계는 반드시 M단계 완료 후 실행
- 독립 단계는 병렬 실행 가능 여부 명시

## 3. 에이전트 구조
- 단일 에이전트 또는 서브에이전트 구성
- (서브에이전트 시) 이름, 역할, 입출력, 트리거 조건

## 4. 구현 스펙 (구조만, 상세 내용은 구현 시 작성)
- 폴더 구조
- CLAUDE.md 핵심 섹션 목록
- 스킬 파일 목록 + 역할 + 트리거 조건
- 스크립트 파일 목록 + 역할
- 데이터 전달 방식
- 주요 산출물 파일 형식

## 5. Claude Code 구현 가이드
- 구현 순서
- 단계별 확인 포인트
```

## 설계 원칙

### 1. 판단과 코드의 역할 분리 — 설계의 핵심

**원칙: 정답이 하나로 정해지면 코드, 판단이 필요하면 AI.**

| AI(에이전트)가 직접 수행 | 코드(스크립트)로 처리 |
|------------------------|---------------------|
| 분류, 의사결정, 우선순위 판단 | 파일 읽기/쓰기, 데이터 파싱 |
| 품질 평가, 정성적 분석 | 외부 API 호출 (정해진 엔드포인트) |
| 컨텍스트 기반 추론 | 반복 처리, 집계, 계산 |
| 자연어 생성/요약 | 포맷 변환, 유효성 검사 |
| 모호한 입력 해석 | 정적 분석, 테스트 실행 |

**왜 중요한가:**
- AI에게 "파일 100개를 읽어서 파싱해"라고 시키면 느리고, 비싸고, 불안정하다. 스크립트가 1초에 끝낼 일이다.
- 반대로 "이 고객 문의가 환불 요청인지 일반 질문인지 판단해"는 코드로 하기 어렵다.
- 이 구분을 설계 단계에서 해두면 구현이 훨씬 안정적이다.

워크플로우의 각 단계에 "담당: AI" 또는 "담당: 코드"를 반드시 명시한다.

---

### 2. 검증 패턴

각 워크플로우 단계마다 "이 단계가 잘 끝났는가"를 확인하는 방법을 정한다.
산출물의 성격에 맞는 검증 방법을 선택한다:

| 유형 | 적용 대상 | 예시 |
|------|----------|------|
| **스키마 검증** | 구조화된 산출물 | JSON 필수 필드 존재, 타입 체크 |
| **규칙 기반** | 정량 기준이 있는 작업 | 항목 수 일치, 글자 수 범위, 포맷 준수 |
| **LLM 자기 검증** | 정성적 판단 작업 | 요약 품질, 톤 적합성, 누락 여부 |
| **사람 검토** | 고위험 최종 산출물 | 외부 발송 문서, 중요 의사결정 |

설계서 작성 시 각 단계마다 명시할 것:
- **성공 기준**: 이 단계의 산출물이 충족해야 할 조건
- **검증 방법**: 위 유형 중 어떤 방식으로 확인하는가

---

### 3. 실패 처리

| 방식 | 사용 시점 | 비고 |
|------|----------|------|
| **자동 재시도** | 단순 누락/형식 오류 | 최대 재시도 횟수를 반드시 명시 (예: 최대 2회) |
| **에스컬레이션** | 판단 불확실, 기준 자체가 모호 | 사람에게 확인 요청 |
| **스킵 + 로그** | 선택적 단계, 전체 흐름에 영향 없음 | 사유를 로그에 기록 |

워크플로우의 각 단계에서 "실패 시" 컬럼에 위 3가지 중 하나를 지정한다.
어떤 실패 유형에 어떤 방식을 쓰는지까지 정해야 구현 시 애매하지 않다.

---

### 4. 에이전트 구조 선택

**기본값: 단일 에이전트.**

서브에이전트로 분리하는 조건:
- 지침이 길어서 항상 로드하면 컨텍스트 윈도우가 비효율적일 때
- 독립적인 작업 블록이 명확히 구분되고, 각각 다른 도메인 지식이 필요할 때

서브에이전트 사용 시 규칙:
- 메인 에이전트(CLAUDE.md)가 오케스트레이터 역할
- 서브에이전트 간 직접 호출 금지 → 반드시 메인을 통해 조율
- AGENT.md에 역할, 트리거 조건, 입출력, 참조 스킬 명시

---

### 5. 스킬 vs 서브에이전트

| | 스킬 (Skill) | 서브에이전트 (Subagent) |
|--|-------------|----------------------|
| **크기** | 작음. 도구/기능 단위 | 큼. 역할/책임 단위 |
| **공유** | 여러 에이전트가 공유 가능 | 특정 워크플로우 전용 |
| **예시** | `file-parser`, `api-caller`, `validator` | `code-reviewer`, `report-generator` |
| **분리 시점** | 반복 사용되는 기능이 있을 때 | 독립 작업 블록 + 별도 도메인 지식 필요 시 |

---

### 6. 데이터 전달 패턴

| 방식 | 사용 시점 | 예시 |
|------|----------|------|
| **파일 기반** | 데이터가 크거나 구조화된 경우 | `/output/step1_result.json` |
| **프롬프트 인라인** | 데이터가 작고 단순한 경우 (수 줄 이내) | 직접 프롬프트에 포함 |

권장: 중간 산출물은 `/output/`에 저장하고 파일 경로만 전달.
이유: 인라인으로 넘기면 컨텍스트 윈도우를 소모하고, 데이터가 크면 잘릴 수 있다.

### 7. 폴더 구조 표준

```
/project-root
  ├── CLAUDE.md                        # 메인 에이전트 지침
  ├── /.claude
  │   ├── /skills/<skill-name>
  │   │   ├── SKILL.md                 # 스킬 정의 (역할, 트리거 조건)
  │   │   ├── /scripts                 # 결정론적 도구 (파싱, API 호출 등)
  │   │   └── /references              # (선택) 도메인 지식, API 가이드
  │   └── /agents/<subagent-name>
  │       └── AGENT.md                 # 서브에이전트 정의 (역할, 입출력)
  ├── /output                          # 산출물 + 중간 결과
  └── /docs                            # PRD, 설계서 등 참조 문서
```

주의: CLAUDE.md, AGENT.md, SKILL.md의 **상세 내용은 설계서에 쓰지 않는다.** 설계서는 구조와 역할 정의까지만. 실제 내용은 구현 시 작성한다.

---

### 8. 완료 조건 정의

설계서의 "작업 컨텍스트" 섹션에 반드시 완료 조건을 명시한다.

좋은 완료 조건 예시:
- "워크플로우 전체가 입력 → 최종 출력까지 에러 없이 통과하면 끝"
- "10건의 테스트 입력에 대해 정확도 90% 이상이면 끝"
- "모든 단계가 성공 기준을 통과하고, 최종 산출물이 `/output/`에 생성되면 끝"

나쁜 완료 조건 예시:
- "잘 동작하면 끝" (기준 없음)
- "사용자가 만족하면 끝" (측정 불가)

## Claude Code에 설계서를 전달하는 방법

1. 설계서를 `/docs/` 폴더에 저장한다.
2. Claude Code에서 다음과 같이 시작한다:

```
/docs/ 폴더의 설계서를 읽고 구현해줘.
워크플로우 1단계부터 시작하고, 각 단계가 끝날 때마다 성공 기준에 맞는지 확인해줘.
통과해야 다음 단계로 넘어가.
```

3. 각 단계가 끝날 때마다 성공 기준 확인 → 통과하면 다음 단계.
4. 새 세션 시작 시: "CLAUDE.md 읽고, 이전까지 N단계까지 완료했으니 N+1단계부터 시작해줘."
